# Component Contracts

Universal rules for how code components translate to Figma component sets. The actual project-specific component inventory is generated by `/figma-suite setup` into `component-contracts.generated.md` (relative within workspace).

---

## Mapping Principles

| Code concept | Figma concept |
|-------------|---------------|
| Component file | Component set on its own page |
| Union type prop (`"primary" \| "secondary"`) | Variant property |
| Boolean prop (`disabled`, `loading`) | Boolean property |
| String prop (`label`, `placeholder`) | Text property |
| ReactNode prop (`icon`, `leftSlot`) | Instance swap property |
| Size prop (`"sm" \| "md" \| "lg"`) | Variant property (size axis) |
| State (hover, pressed, focused) | Variant property (state axis) or interactive component state |
| Conditional rendering | Boolean property toggling layer visibility |
| `className` / style overrides | Not mapped — handled by variable bindings |

---

## Component Analysis Process

When reading a code component to create a Figma counterpart:

### Step 1: Identify the variant axes

Read the component's props/interface and extract:
- **Variant unions** → Figma variant property
- **Size unions** → Figma size property
- **Boolean flags** → Figma boolean properties
- **String props** → Figma text properties
- **ReactNode/slot props** → Figma instance swap properties

### Step 2: Extract visual specs from styling

Map the component's styling (Tailwind classes, CSS modules, styled-components, etc.) to concrete values:
- Heights, widths, min-sizes per variant/size
- Padding and gap per variant/size
- Typography scale per variant/size
- Colors per variant
- Border radius

Every extracted value must map to a token variable — not a raw value.

### Step 3: Map tokens

For each variant, list:
- Fill color → which semantic color variable
- Text color → which semantic color variable
- Stroke → which semantic color variable (if applicable)
- Radius → which radius variable
- Spacing → which spacing variables

### Step 4: Determine variant matrix

Total variants = product of all variant axes. Keep manageable:
- Use **variant properties** for visual differences (variant, size)
- Use **boolean properties** for states (disabled, loading) — avoids exponential matrix
- Avoid more than ~20 variants per component set

## Figma Component Structure

### Naming convention
```
ComponentName/Variant=Value, Size=Value
```

### Auto-layout setup

Every component frame must use auto-layout:

```
Component frame
├── direction: horizontal or vertical (match code layout)
├── padding: bound to spacing variables
├── gap: bound to spacing variables
├── sizing: hug contents (default) — see Sizing Rules below
├── children:
│   ├── Slot instance (INSTANCE_SWAP property, NOT an empty frame)
│   ├── Text layer (TEXT property + all 4 typography variables bound)
│   └── Trailing element (optional)
```

### Sizing rules

**Dynamic-height components** (Card, Dialog, BottomSheet, EmptyState, Toast, Input multiline) must use hug contents on the vertical axis:
```javascript
comp.layoutSizingVertical = "HUG";   // height adapts to content
comp.layoutSizingHorizontal = "FIXED"; // or "HUG" for inline components
```

**Never call `resize(w, h)` on a HUG axis** — it sets FIXED and overrides hug. Only resize the FIXED axis:
```javascript
comp.resize(320, 1); // only width matters when vertical is HUG
```

**Fully hug components** (Button, Badge) use HUG on both axes:
```javascript
comp.layoutSizingHorizontal = "HUG";
comp.layoutSizingVertical = "HUG";
```

**Children that stretch** must set FILL after being appended to their auto-layout parent:
```javascript
parent.appendChild(child);
child.layoutSizingHorizontal = "FILL"; // must come AFTER appendChild
```

### Slots: INSTANCE_SWAP (native Slots not yet available via Plugin API)

Figma's native Slots feature cannot be created programmatically (no `createSlot()`, SLOT type rejected by `componentPropertyReferences`). Use INSTANCE_SWAP as the workaround. After build, the user can manually convert to native Slots in the Figma UI if desired.

Content slots in components (Card content, BottomSheet content, Dialog footer) must be real INSTANCE_SWAP properties:

```javascript
// 1. Create a placeholder component
const placeholder = figma.createComponent();
placeholder.name = "_Slot Placeholder";
placeholder.resize(100, 40);
placeholder.fills = [];

// 2. Instance it inside the parent
const slot = placeholder.createInstance();
parent.appendChild(slot);
slot.layoutSizingHorizontal = "FILL";

// 3. Register as INSTANCE_SWAP property
parent.addComponentProperty("Content", "INSTANCE_SWAP", placeholder.id);
```

**Never use a raw Frame as a slot.** Consumers cannot swap content into frames — only into instance swap properties.

For optional slots, pair with a boolean property:
```javascript
comp.addComponentProperty("Show CTA", "BOOLEAN", true);
comp.addComponentProperty("CTA", "INSTANCE_SWAP", buttonComp.id);
```

### Variable binding checklist

For every component, bind:
- [ ] Fill → Semantic color variable (via `setBoundVariableForPaint`)
- [ ] Text fill → Semantic color variable (via `setBoundVariableForPaint`)
- [ ] Stroke → Semantic color variable (via `setBoundVariableForPaint`, if applicable)
- [ ] Corner radius → Radius variable (all 4 corners: `setBoundVariable("topLeftRadius", var)`, etc.)
- [ ] Padding (top, right, bottom, left) → Spacing variables (`setBoundVariable("paddingTop", var)`, etc.)
- [ ] Gap → Spacing variable (`setBoundVariable("itemSpacing", var)`) — choose the semantically correct spacing token (e.g., tight gap for inline elements, section gap for card spacing)
- [ ] Min height / fixed height → Spacing variable (if applicable)
- [ ] **Typography** → Apply a **Text Style** via `setTextStyleIdAsync()` — see below
- [ ] **VERIFY** all bindings with `node.boundVariables` after setting

**`fontSize`, `fontWeight`, and `lineHeight` are NOT bindable via `setBoundVariable()` on text nodes.** They must be set through Text Styles. Do NOT attempt individual font property binding — it silently fails.

If no suitable variable exists for a property and creating one doesn't make sense, use a raw value and report it as an exception with a reason. Never silently skip a binding.

### Text layer: use Text Styles (MUST)

**Always apply a Text Style via `setTextStyleIdAsync()`.** Individual `setBoundVariable` calls for font properties do NOT work in headless `use_figma`.

Sequence:
1. `await figma.loadFontAsync({ family, style })` — load the font (probe available styles first — names vary by provider: "SemiBold" vs "Semi Bold")
2. `text.fontName = { family, style }` — set static font (required before characters)
3. `text.characters = "..."` — set text content
4. `await text.setTextStyleIdAsync(styleId)` — apply Text Style
5. Set text fill color separately (not part of Text Style):
   ```javascript
   const textPaint = figma.variables.setBoundVariableForPaint(
     figma.util.solidPaint("#18181b"), "color", foregroundVar
   );
   text.fills = [textPaint];
   ```

To verify: `text.textStyleId` should start with `"S:"`. If it does, typography is applied via the style.

### Variable modes on components

Components do NOT automatically use non-default variable modes. If you have Light/Dark modes or other variants, you MUST explicitly set the mode on each component frame:

```javascript
comp.setExplicitVariableModeForCollection(collectionId, modeId);
```

Without this call, all components render using the collection's first mode regardless of context. This is critical for dark mode variants and theme switching.

## Component Composition

Composite components must nest instances of simpler components — never rebuild them from raw frames.

### Principles

1. **Reuse tokens** — every visual property bound to a variable, never raw values
2. **Reuse components** — if a child element matches an existing library component, use an instance
3. **Reuse text styles** — every text layer has all 4 typography properties bound individually

### Build order

Components must be built in dependency order:

```
Tier 1 (no library dependencies):  primitives — icons, badges, switches, spinners
Tier 2 (uses Tier 1):              interactive — buttons, inputs, controls
Tier 3 (uses Tier 1-2):            containers — cards, list rows, toasts, empty states
Tier 4 (uses Tier 1-3):            overlays — dialogs, sheets, navigation bars
```

Never build a higher-tier component before its dependencies exist.

The actual tiers for your project are determined by `/figma-suite setup`.

### How to nest

When composing component B inside component A:

1. **Create an instance** of component B inside A's frame — never detach or flatten
2. **Expose B's key properties** through A's component properties:
   - Use **instance swap** to let A's consumer pick which variant of B to use
   - Use **boolean property** on A to show/hide B (e.g., "Show CTA")
   - Use **text property** on A that maps to B's text property (e.g., A's "CTA Label" → B's "Label")
3. **Constrain B within A's auto-layout** — B should fill or hug as appropriate, not use fixed position

### Common composition patterns

**Empty state with CTA:**
```
EmptyState (auto-layout: vertical, center)
├── [Icon/illustration instance] (instance swap)
├── Title text (text property, typography variable-bound)
├── Description text (text property, typography variable-bound)
└── [Button instance] (boolean: "Show CTA", instance swap for variant)
```

**Dialog with actions:**
```
Dialog (auto-layout: vertical)
├── Title text (text property, typography variable-bound)
├── Body text (text property, typography variable-bound)
├── [Button instance — primary] (instance swap)
└── [Button instance — secondary] (boolean: "Show Secondary", instance swap)
```

**Container with list rows:**
```
Card (auto-layout: vertical, padding + radius variable-bound)
├── [ListRow instance]
├── Separator (1px, fill: semantic color variable)
├── [ListRow instance]
├── Separator
└── [ListRow instance]
```

These are generic patterns. The actual component names and structures are project-specific and generated by `/figma-suite setup`.

## Component Property Keys

**Property names are NOT simple strings.** TEXT, BOOLEAN, and INSTANCE_SWAP properties have a `#uid` suffix (e.g., `"Label#4:0"`, `"Show Icon#2:33"`). Only VARIANT properties are plain names (e.g., `"Size"`).

- **Always capture the return value** from `addComponentProperty()` — it returns the full key with suffix
- **Never hardcode or guess keys** — inspect `componentPropertyDefinitions` to discover them
- **Using the wrong key produces silent failures** — the property appears to exist but isn't linked

```javascript
// Discover existing property keys
const propDefs = comp.componentPropertyDefinitions;
for (const [key, def] of Object.entries(propDefs)) {
  // key = "Label#4:0", def.type = "TEXT"
  // key = "Show Icon#2:33", def.type = "BOOLEAN"
  // key = "Size", def.type = "VARIANT" (no suffix)
}
```

## Verification Script

Run this script via `use_figma` after creating or modifying any component. It walks every node and checks for unbound properties, missing component properties, and unapplied text styles. Pass the component or component set node ID.

**What it checks:**
- SOLID fills and strokes → must be variable-bound
- Corner radius → must be variable-bound
- Padding and gap on auto-layout frames → must be variable-bound
- Text layers → must have a Text Style applied (NOT individual font bindings — `fontSize`, `fontWeight`, `lineHeight` are NOT bindable via `setBoundVariable`)
- Text layers → must have fill variable-bound
- Text layers → must be linked to a TEXT component property
- Instance nodes → must be linked to an INSTANCE_SWAP component property

```javascript
const nodeId = "COMPONENT_OR_SET_ID"; // replace with actual ID
const root = await figma.getNodeByIdAsync(nodeId);
if (!root) return { error: "Node not found" };

const violations = [];

const variants = root.type === "COMPONENT_SET" ? [...root.children] : [root];

for (const variant of variants) {
  const allNodes = variant.findAll(() => true);

  for (const node of allNodes) {
    const bv = node.boundVariables || {};
    const path = `${variant.name}/"${node.name}"`;

    // Check SOLID fills are variable-bound (skip gradients, images)
    if (node.fills && node.fills.length > 0 && node.fills[0].type === "SOLID") {
      if (!bv.fills) {
        violations.push(`${path}: SOLID fill not variable-bound`);
      }
    }

    // Check SOLID strokes
    if (node.strokes && node.strokes.length > 0 && node.strokes[0].type === "SOLID") {
      if (!bv.strokes) {
        violations.push(`${path}: SOLID stroke not variable-bound`);
      }
    }

    // Check radius (safe type check — may be Symbol)
    if (typeof node.topLeftRadius === "number" && node.topLeftRadius > 0) {
      if (!bv.topLeftRadius) {
        violations.push(`${path}: corner radius not variable-bound`);
      }
    }

    // Check padding and gap (auto-layout frames only)
    if (node.layoutMode && node.layoutMode !== "NONE") {
      if (typeof node.paddingTop === "number" && node.paddingTop > 0 && !bv.paddingTop) {
        violations.push(`${path}: padding not variable-bound`);
      }
      if (typeof node.itemSpacing === "number" && node.itemSpacing > 0 && !bv.itemSpacing) {
        violations.push(`${path}: gap (itemSpacing) not variable-bound`);
      }
    }

    // Check text layers
    if (node.type === "TEXT") {
      // Text Style applied? (NOT individual font bindings — they don't work in headless)
      const hasStyle = typeof node.textStyleId === "string" && node.textStyleId.startsWith("S:");
      if (!hasStyle) {
        violations.push(`${path}: no Text Style applied (do NOT use setBoundVariable for font properties)`);
      }

      // Text fill variable-bound?
      if (!bv.fills) {
        violations.push(`${path}: text fill not variable-bound`);
      }

      // Linked to a TEXT component property?
      const refs = node.componentPropertyReferences || {};
      if (!refs.characters) {
        violations.push(`${path}: not linked to a TEXT component property`);
      }
    }

    // Check instances are linked to INSTANCE_SWAP
    if (node.type === "INSTANCE") {
      const refs = node.componentPropertyReferences || {};
      if (!refs.mainComponent) {
        violations.push(`${path}: instance not linked to INSTANCE_SWAP property`);
      }
    }
  }
}

return {
  component: root.name,
  violationCount: violations.length,
  violations: violations.slice(0, 30),
  hasMoreViolations: violations.length > 30
};
```

**How to use the output:**

- **Violations (count > 0):** Review each one. Fix genuine violations before proceeding.
- **Some violations may be valid exceptions** — e.g., a decorative fill that shouldn't be bound, an internal layout helper that shouldn't expose a TEXT property. If so, acknowledge it as an intentional skip with a reason and move on.
- **Re-run the script** after fixes to confirm violations are resolved.
- **Report all exceptions** to the user so they can see every decision that was made.
